## Vision

### 10x Check
The 10x version turns WardBulletin into an SEO-powered content hub — programmatic pages for every hymn, every meeting type, localized guide content in 9 languages, and deep links from content → editor with templates pre-loaded. Each guide page becomes a conversion funnel, not just content.

## Accepted Scope (added to this plan)
- StaticPageLayout component (DRY wrapper for static pages)
- Full i18n translation of guide page content into 9 languages

- Template pre-population with today's date, ward name, leadership from localStorage
- Build-time sitemap generation from route definitions
- Canonical URL normalization: wardbulletin.com (no www) as canonical everywhere
- Unsaved changes confirmation when template applied via deep link
- Critical test coverage (builtInTemplates, songService, template merge, deep links, sitemap)

## Design Decisions (from /plan-design-review)
- Guide pages: icons on H2 headings, numbered steps, tip callout boxes, table of contents at top
- Guide pages: header image from existing LDS artwork (calling_the_fishermen, go_ye_therefore, christ_ordaining)
- Template selection: toast notification via react-toastify ("Sacrament Meeting template applied")
- Deep link: dismissible onboarding banner, persisted via localStorage ("don't show again")
- Responsive: grid-cols-1 sm:grid-cols-2 for template modal built-in grid, wrapping footer links
- CTAs: contextually relevant copy per guide page ("Start Building Your Bulletin", "Try a Template Now", "Create Your Program")
